¿Qué es Git?

Git es un sistema de control de versiones distribuido y de código abierto; es rápido, eficiente con grandes proyectos y posee un sistema de ramificación (branching) para desarrollo no lineal. Fue creado por Linus Torvalds, fundador de Linux.

Un control de versiones es un sistema que registra los cambios realizados en un archivos a lo largo del tiempo, permitiendo recuperar cualquier versión trabajada.

Git almacena y maneja información como conjunto de copias instantáneas de un sistema de archivos, como archivos miniatura, que permiten guardar una referencia en ellos. Además, la mayoría de las operaciones en Git se hacen con archivos y recursos locales, por ello evita el problema de la velocidad de red. Finalmente, su integridad radica en la suma de comprobaciones antes de ser almacenado, y la identificación de cada cambio.

Flujo de trabajo básico en Git

  • Modificas una serie de archivos en tu directorio de trabajo.
  • Preparas los archivos, añadiéndolos a tu área de preparación.
  • Confirmas los cambios, lo que toma los archivos tal y como están en el área de preparación y almacena esa copia instantánea de manera permanente en tu directorio de Git.

Puedes leer más sobre este sistema de control de versiones desde: https://git-scm.com/book/es/v2

¿Qué es GitHub?

GitHub es un proveedor de alojamiento para repositorios Git, permitiendo proyectos de código abierto, seguimiento de fallos, revisión de código, etc. No es parte directa de Git, pero si es totalmente compatible y útil su uso en conjunto, para facilitar el trabajo colaborativo.

Acciones principales de interacción en GitHub

  1. Crear un repositorio o clonar uno existente. Los repositorios serán los lugares donde se alojan tus proyectos.
  2. Bifurcación: El proceso de copiar el código de otra persona del repositorio para modificarlo.
  3. Pull: Cuando haya terminado de hacer cambios en el código de otra persona, puede compartirlos con el propietario original a través de una pull request.
  4. Fusión: Los propietarios pueden añadir nuevos cambios a sus proyectos a través de una fusión, y dar crédito a los contribuyentes que los han sugerido.

Integración git y GitHub

Para que se puedan integrar git con Github sigue los siguientes pasos, luego de la instalación de git.

  1. Crea un usuario o inicia sesión en Github, recuerda que tienes la opción gratuita además de opciones pagas.
  2. Crea o clona un repositorio de GitHub a tu cuenta (En el punto siguiente se verá como clonar y trabajar en git desde local).
    1. Para crear un repositorio: https://git-scm.com/book/es/v2/GitHub-Creaci%C3%B3n-y-configuraci%C3%B3n-de-la-cuenta
  3. Pasos para trabajar en Git, clonar repositorios y modificar archivos
    1. Abrir la terminal GIT CMD
    2. Elegir ubicación para la carpeta donde va a alejarse el sitio en local:
      3. cd ubicación-deseada
    3. Clonar el repositorio en la PC:
      4. git clone urldelrepositoriodegit

    4. Ahora puedes trabajar en el archivo que desees del repositorio

    5. Una vez realizadas y guardadas las modificaciones, desde la terminal puedes observar el estado de las modificaciones:
      git status
    6. Ahora, debes añadir los archivos modificados que serán subidos/actualizdos en el repositorios:
      7. git add nombredelarchivo
    7. Siempre es conveniente escribir un mensaje sobre los cambios realizados:
      8. git commit -m “breve descripcion de cambios”
    8. Traer cambios hechos por otros usuarios al mismo tiempo:
      9. git pull
    9. Subir las modificaciones hechas en tu local al repositorio de github:
      10. git push
